# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Commands

```bash
# Start Metro bundler
npm start

# Run on device/simulator
npm run ios        # requires Mac + Xcode
npm run android
npm run web

# Tests
npm test                    # run all tests
npm run test:watch          # watch mode
npm run test:coverage       # with coverage report in /coverage

# Run a single test file
npx jest src/carrier/hooks/__tests__/useVendorRequestsList.test.ts

# Native setup (only when plugins or deps change)
npx expo prebuild           # regenerate ios/ and android/
cd ios && pod install && cd ..

# Production builds (EAS)
eas build --platform ios
eas build --platform android
eas submit
```

## Architecture

**Expo bare workflow** — `ios/` and `android/` exist in the repo but are generated by `expo prebuild`. Never edit them directly. Any change to plugins in `app.json` requires a new prebuild + native rebuild.

### Three user roles

| Role | Directory | Who they are |
|------|-----------|-------------|
| **Carrier** | `src/carrier/` | Dispatcher / fleet manager |
| **Driver** | `src/driver/` | Truck driver |
| **Finder** | `src/finder/` | Shipper / cargo owner ("Finder" is the internal code term) |

Each role has its own screens, hooks, and navigator. Code that crosses roles lives in `src/core/` (shared infrastructure) or `src/modules/` (cross-role feature modules).

### Cross-role modules (`src/modules/`)

| Module | What it does | Roles |
|--------|-------------|-------|
| `inspections/` | Pre/post-trip vehicle inspections, PDF generation, offline sync | Carrier + Driver |
| `contracts/` | Digital signature flow for freight contracts | Carrier + Finder |
| `projects/` | Multi-leg operations (23+ hooks) | Carrier + Finder |
| `vehicleExpenses/` | Fuel/repair expense tracking | Driver |
| `aiSupport/` | In-app AI support assistant (context-aware by role) | All |

### `src/core/` — shared infrastructure

| Subdirectory | Purpose |
|-------------|---------|
| `chat/` | IMChat system (`IMChatScreen`, `firebaseChatClient`) |
| `dopebase/` | UI component system, theme, i18n, ActionSheet |
| `firebase/` | Firebase init (`config.ts`), Firestore/Auth/Functions setup |
| `components/` | Shared components: `MapViewBase`, `AnimatedMarker`, `RouteMap`, `MultiSelectSheet` |
| `onboarding/` | Auth flow + `AuthProvider` |
| `notifications/` | Push handlers + `InAppNotificationProvider` |
| `navigation/` | `RootNavigation.ts` (global navigation ref) |
| `media/` | `storageAPI.ts` — Firebase Storage upload/download |

### State management

**Redux** (`src/redux/`) with 6 slices using `legacy_createStore` (not RTK yet):

| Slice | Key state |
|-------|-----------|
| `auth` | Firebase auth state, `setUserData` / `logOut` |
| `trip` | Origin, destination, cargo, ETA, distance for a trip under construction |
| `finderRequestPackage` | Multi-leg request builder (add/update/remove routes) |
| `ride` | Vehicle/category selection |
| `bottomSheet` | Snap points control |
| `operationSheet` | Operation panel data |

**Context providers** (wrapping order in `App.tsx`): `ConfigProvider` → `AuthProvider` → `ProfileAuthProvider` → `DopebaseProvider` → `TranslationProvider` → `ActionSheetProvider` → `InAppNotificationProvider` → `SafeAreaProvider`.

### Firebase collections

| Collection | Purpose |
|-----------|---------|
| `channels` / `messages_live` | Chat messages (real-time) |
| `requests` | Cargo requests (Finder creates, Carrier reads) |

### External services

| Service | Purpose | Location |
|---------|---------|---------|

## Code patterns

**Hook contract** — all domain hooks return `{ data, loading, error }` and use `try/catch/finally` with `setLoading`. Firestore listeners always return the unsubscribe function from `useEffect`.

**File colocation** — each screen/component folder contains `index.tsx`, `styles.ts`, and optionally `types.ts`. Barrel exports from each feature directory via `index.ts`.

**Theme/spacing** — always via hooks: `const { theme } = useTheme()` and `const { spacing } = useSpacing()`. Never hardcode values.

**i18n** — use `localized()` from `useTranslations()`, never raw strings in UI.

## Testing

**Use `renderHook` from `@testing-library/react-native`** — not from `@testing-library/react-hooks` (deprecated, React 16/17 only).

**Firestore mock pattern:**
```typescript
mockOnSnapshot.mockImplementation((ref, onNext) => {
  onNext({ docs: [...] });
  return mockUnsubscribe;
});
await waitFor(() => expect(result.current.loading).toBe(false));
act(() => unmount());
expect(mockUnsubscribe).toHaveBeenCalledTimes(1);  // verify cleanup
```

Use `jest.clearAllMocks()` in `beforeEach`. Tests for hooks with Firestore live under `src/<role>/hooks/__tests__/`.

**Coverage priority:** role hooks (`carrier/`, `driver/`, `finder/`) → shared modules (`modules/`) → core components (`core/`).

**TDD rule:** write the failing test first (red), then implement (green).

## Key config locations

| What | Where |
|------|-------|
| Firebase SDK init | `src/core/firebase/config.ts` |
| App-level config (tabs, drawer menus, thresholds) | `src/config/` via `ConfigProvider` |
| Google Maps API key | `app.json` → `android.config.googleMaps.apiKey` |

| Expo / EAS project IDs | `app.json` → `expo.extra` and `eas.json` |

## Known gotchas

**`babel.config.js`** — `reanimated: false` is isolated to the `isTest` branch. The non-test branch must return `presets: ['babel-preset-expo']` as a plain string (not `['babel-preset-expo', {}]`). Passing an explicit `{}` options object triggers `babel-preset-expo` v55 to resolve `react-refresh/babel`, breaking iOS and Metro builds.

**`react-test-renderer`** — pinned to `19.2.0` to match `react@19.2.0`. Update both together.

**`expo prebuild --clean`** — deletes `ios/` and `android/` entirely. Firebase credential files (`GoogleService-Info.plist`, `google-services.json`) are lost and must be restored manually from a secure location before rebuilding.

**Files with "copy" in the name** — ~20 duplicate files (`IMChatScreen copy.tsx`, etc.) scattered across `src/`. These are dev artifacts, not canonical files. Never import from them.

**Large files (handle with care)** — read the full file before editing; state is often shared across 500+ lines:
