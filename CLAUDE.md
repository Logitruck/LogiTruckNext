## Architecture

**Expo bare workflow** ‚Äî `ios/` and `android/` exist in the repo but are generated by `expo prebuild`. Never edit them directly. Any change to plugins in `app.json` requires a new prebuild + native rebuild.

### Three user roles

| Role | Directory | Who they are |
|------|-----------|-------------|
| **Carrier** | `src/carrier/` | Dispatcher / fleet manager |
| **Driver** | `src/driver/` | Truck driver |
| **Finder** | `src/finder/` | Shipper / cargo owner ("Finder" is the internal code term) |

Each role has its own screens, hooks, and navigator. Code that crosses roles lives in `src/core/` (shared infrastructure) or `src/modules/` (cross-role feature modules).

### Cross-role modules (`src/modules/`)

| Module | What it does | Roles |
|--------|-------------|-------|
| `inspections/` | Pre/post-trip vehicle inspections, PDF generation, offline sync | Carrier + Driver |
| `contracts/` | Digital signature flow for freight contracts | Carrier + Finder |
| `projects/` | Multi-leg operations (23+ hooks) | Carrier + Finder |
| `vehicleExpenses/` | Fuel/repair expense tracking | Driver |
| `aiSupport/` | In-app AI support assistant (context-aware by role) | All |

## Code patterns

**Hook contract** ‚Äî all domain hooks return `{ data, loading, error }` and use `try/catch/finally` with `setLoading`. Firestore listeners always return the unsubscribe function from `useEffect`.

**File colocation** ‚Äî each screen/component folder contains `index.tsx`, `styles.ts`, and optionally `types.ts`. Barrel exports from each feature directory via `index.ts`.

**Theme/spacing** ‚Äî always via hooks: `const { theme } = useTheme()` and `const { spacing } = useSpacing()`. Never hardcode values.

**i18n** ‚Äî use `localized()` from `useTranslations()`, never raw strings in UI.

## Testing

**Use `renderHook` from `@testing-library/react-native`** ‚Äî not from `@testing-library/react-hooks` (deprecated, React 16/17 only).

**Firestore mock pattern:**
```typescript
mockOnSnapshot.mockImplementation((ref, onNext) => {
  onNext({ docs: [...] });
  return mockUnsubscribe;
});
await waitFor(() => expect(result.current.loading).toBe(false));
act(() => unmount());
expect(mockUnsubscribe).toHaveBeenCalledTimes(1);  // verify cleanup
```

Use `jest.clearAllMocks()` in `beforeEach`. Tests for hooks with Firestore live under `src/<role>/hooks/__tests__/`.

**Coverage priority:** role hooks (`carrier/`, `driver/`, `finder/`) ‚Üí shared modules (`modules/`) ‚Üí core components (`core/`).

**TDD rule:** write the failing test first (red), then implement (green).

## Known gotchas

**`babel.config.js`** ‚Äî `reanimated: false` is isolated to the `isTest` branch. The non-test branch must return `presets: ['babel-preset-expo']` as a plain string (not `['babel-preset-expo', {}]`). Passing an explicit `{}` options object triggers `babel-preset-expo` v55 to resolve `react-refresh/babel`, breaking iOS and Metro builds.

**`react-test-renderer`** ‚Äî pinned to `19.2.0` to match `react@19.2.0`. Update both together.

**`expo prebuild --clean`** ‚Äî deletes `ios/` and `android/` entirely. Firebase credential files (`GoogleService-Info.plist`, `google-services.json`) are lost and must be restored manually from a secure location before rebuilding.

**Files with "copy" in the name** ‚Äî ~20 duplicate files (`IMChatScreen copy.tsx`, etc.) scattered across `src/`. These are dev artifacts, not canonical files. Never import from them.

**Large files (handle with care)** ‚Äî read the full file before editing; state is often shared across 500+ lines:
